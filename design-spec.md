# Slidememos 视觉设计规范

## 1. 品牌定位

**产品名称**: Slidememos
**产品类型**: 浏览器侧边栏笔记扩展
**核心价值**: 轻量、快速、沉浸式的碎片信息收集工具
**目标用户**: 知识工作者、研究人员、内容创作者

**品牌关键词**:
- 轻盈 (Lightweight)
- 流畅 (Fluid)
- 专注 (Focused)
- 智能 (Smart)

## 2. 扩展图标 (Extension Icon)

### 2.1 尺寸要求

| 尺寸 | 用途 | 文件名 |
|------|------|--------|
| 16x16 px | 浏览器工具栏 | icon16.png |
| 48x48 px | 扩展管理页面 | icon48.png |
| 128x128 px | Chrome 商店展示 | icon128.png |

### 2.2 格式要求
- **格式**: PNG
- **背景**: 透明
- **色彩模式**: sRGB
- **导出**: 需要 @1x 和 @2x 版本（高清屏适配）

### 2.3 设计方向建议

**概念 A: 侧边栏 + 笔记**
- 抽象的侧边栏形态（竖向矩形）
- 内含笔记/卡片元素
- 体现"滑入"的动感

**概念 B: 便签/卡片堆叠**
- 多张卡片层叠效果
- 暗示快速收集、堆积
- 简洁的几何形状

**概念 C: 书签 + 闪电**
- 书签形态代表收藏
- 闪电元素代表快速
- 体现"一键保存"的核心功能

### 2.4 色彩建议

**主色调** (可选方案):
```
方案 1: 蓝色系 - #3B82F6 (信任、专业)
方案 2: 紫色系 - #8B5CF6 (创意、智能)
方案 3: 绿色系 - #10B981 (效率、成长)
```

**注意事项**:
- 图标需在浅色/深色工具栏上都清晰可辨
- 避免过多细节，16px 下仍需可识别
- 考虑与 Chrome 原生图标的视觉协调

## 3. 侧边栏 UI 设计

### 3.1 布局规范

```
┌─────────────────────┐
│  Logo + 搜索框      │  <- Header: 固定
├─────────────────────┤
│  编辑器区域         │  <- 可折叠
├─────────────────────┤
│                     │
│  卡片列表           │  <- 可滚动
│  (虚拟滚动)         │
│                     │
└─────────────────────┘
```

**侧边栏宽度**: 360-400px (Chrome Side Panel 标准)

### 3.2 卡片设计

每张 Memo 卡片需包含:
- 类型图标 (📝 文本 / 🔗 链接 / 💻 代码)
- 标题 (单行，截断)
- 内容预览 (2-3 行)
- 时间戳
- 操作按钮 (hover 显示)

**卡片状态**:
- Default: 静止状态
- Hover: 轻微上浮 + 阴影增强
- Active: 边框高亮
- New: 闪烁动效 (刚创建时)

### 3.3 动效规范

| 动效 | 时长 | 缓动函数 | 场景 |
|------|------|----------|------|
| 卡片 Hover | 300ms | ease-out | 鼠标悬停 |
| 卡片创建 | 300ms | ease-out | 新建 Memo |
| 卡片删除 | 200ms | ease-in | 删除 Memo |
| Toast 弹出 | 300ms | ease-out | 操作反馈 |
| 搜索高亮 | 150ms | linear | 关键词匹配 |

### 3.4 Dark Mode

需同时提供浅色/深色主题设计:

| 元素 | Light Mode | Dark Mode |
|------|------------|-----------|
| 背景 | #FFFFFF | #1A1A1A |
| 卡片背景 | #F5F5F5 | #2A2A2A |
| 主文字 | #1A1A1A | #FFFFFF |
| 次要文字 | #666666 | #A0A0A0 |
| 边框 | #E0E0E0 | #3A3A3A |
| 主色调 | #3B82F6 | #60A5FA |
